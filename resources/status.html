Transcript Processing Delays

February 28, 2026 — Resolved

Between 14:12 and 15:47 UTC, some users experienced delays of up to 10 minutes in transcript processing. The issue was caused by a temporary spike in processing queue depth following a surge in concurrent uploads. We scaled our processing infrastructure and the backlog cleared within 30 minutes. No data was lost.

Duration: 1 hour 35 minutes
Impact: Delayed transcript delivery for approximately 12% of active users
Root cause: Processing queue capacity exceeded during peak usage

API Intermittent 503 Errors

January 15, 2026 — Resolved

Between 09:30 and 10:15 UTC, a small number of API requests returned 503 Service Unavailable errors. The issue was traced to a misconfigured load balancer rule deployed during a routine infrastructure update. The configuration was rolled back and all services returned to normal.

Duration: 45 minutes
Impact: Approximately 0.3% of API requests affected
Root cause: Load balancer misconfiguration during maintenance
